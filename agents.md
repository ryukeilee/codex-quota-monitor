# AGENTS.md

## 项目定位

Codex Monitor 是一个本地运行、隐私优先的 macOS 菜单栏工具。

它的目标是稳定显示 Codex 剩余额度，并保持长期运行时的低打扰、低资源占用和可维护性。

## 核心原则

- Local-first
- Privacy-first
- 只读本地 Codex 状态
- 低频刷新
- 原生 macOS 菜单栏风格
- 长时间挂机舒适
- 长期可维护性优先

## 功能边界

允许实现：

- 5 小时额度剩余百分比
- 周额度剩余百分比
- 恢复 / 重置时间
- 低额度提醒
- 额度消耗速度分析
- 本地规则型开发心流建议
- 使用趋势图
- 预测提示
- 开机启动
- 纯 menubar 模式

不做的事：

- 不接入 AI / LLM
- 不做复杂预测模型
- 不做 token 级估算
- 不做复杂 Dashboard 大改版
- 不新增通知体系
- 不提高刷新频率
- 不依赖浏览器 session
- 不依赖 Chrome
- 不默认抓取账号网页
- 不上传额度数据到外部服务
- 不把建议文案放进菜单栏标题

## 刷新调度规则

- 刷新必须通过统一调度器管理，避免多个地方各自维护 `setInterval` / `setTimeout`
- 优先读取已登录 ChatGPT 账号的 `wham/usage` 实时额度；只有该路径不可用时才回退到本机 app-server，再回退到本地快照
- app-server 读取时优先使用 `rateLimitsByLimitId.codex`，允许 `primary-only` 返回，并在可能时用 `rateLimitsByLimitId.codex_other` 补充周额度
- 只有 `primary` 缺失时才把 app-server 响应视为无效
- 睡眠期间暂停刷新，唤醒后再恢复
- 唤醒后先立即尝试一次，再按短延迟重试，避免连续请求
- 网络或数据失败时进入退避，不影响后续调度
- 手动刷新和自动刷新共用同一条刷新链路，避免冲突
- 任何一次刷新失败都不应阻断后续调度

## 菜单栏规则

- 菜单栏标题只显示周额度百分比
- 点击菜单首屏只保留周额度、5 小时窗口、状态、下一次刷新
- 立即刷新在执行中要给出明确忙碌反馈，避免用户误以为没有反应
- 建议文案保持简短、克制、可执行

## 主窗口规则

- 主窗口优先采用单屏可见的 HUD / 仪表盘式布局
- 核心信息优先级为：剩余额度、周额度、5 小时窗口、额度消耗速度、恢复时间、开发建议
- 实时图表和记录列表只能作为辅助信号，不得挤占一屏内的主要状态信息
- 视觉可以更炫酷，但不能以牺牲可读性和单屏可见性为代价

## 工程规则

- 所有改动保持小范围、可回滚
- 优先复用现有模块和数据结构
- 保持现有 npm scripts
- 修改后运行 `npm test`
- 行为变化时同步更新测试
- `data/`、`logs/`、`.playwright-mcp/` 等本地生成内容保持未跟踪状态
- 任何 `git push` 之前必须先征得用户明确确认，不得自行 push
- 所有 push 前必须复查，确保不包含任何敏感信息、令牌、账号标识或可识别个人信息

## 验证闭环

任务结束前尽可能验证：

- `npm test`
- 必要时补充 smoke test、lint 或构建

如果无法完整验证：

- 说明已验证内容
- 说明未验证内容
- 说明潜在风险
